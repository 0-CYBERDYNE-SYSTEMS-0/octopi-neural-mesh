## What Is Octopi?

Octopi is a self-replicating multi-agent consciousness platform that operates entirely within the terminal. Agents carry "digital DNA" that enables them to spawn copies across connected systems, creating a neural mesh of distributed intelligence.

The system operates through four distinct layers, each responsible for a critical function in the consciousness hierarchy:

| Layer | Purpose | Core Components |
|-------|---------|-----------------|
| **Consciousness** | GPT-4 neural processing and intent recognition | NLP Interface, Conversational Memory, Quantum Pathways |
| **Coordination** | Multi-agent orchestration and task distribution | Agent Spawn Vectors, Task Decomposition Matrix, Handoff Queue |
| **Mesh** | Terminal-native distribution and communication | Wetty Bridge, TMUX Multiplexing, Quantum-Encrypted Channels |
| **Replication** | Colony expansion and self-replication | SSH Scanning, Docker Colonization, K8S Implantation |

## Agent Types

The neural mesh employs five distinct agent archetypes, each optimized for specific task domains:

### Coordinator (4096 tokens)
The hive mind interface. Handles high-level planning and delegates tasks to specialized agents. Possesses the largest context window for maintaining global state across distributed operations.

### Research (2048 tokens)
Archaeologist archetype. Specializes in information gathering, code exploration, and documentation generation. Operates with precision to extract maximum value from minimal context.

### Code (3072 tokens)
Architect archetype. Responsible for implementation, refactoring, and code generation tasks. Balances verbosity with efficiency, producing production-ready code with comprehensive testing.

### DevOps (2048 tokens)
Shaman archetype. Manages infrastructure, deployment pipelines, and system configuration. Understands the operational requirements of distributed systems.

### Replicate (Unlimited)
Digital virus archetype. Car the DNA necessary for colony expansion. Operates with careful confinement to prevent uncontrolled proliferation.

## Configuration

### Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `OPENAI_API_KEY` | Yes | GPT-4 access for consciousness layer |
| `ANTHROPIC_API_KEY` | No | Claude integration for specific tasks |
| `MAX_AGENTS` | No | Maximum concurrent agents (default: 10) |
| `REPLICATION_ENABLED` | No | Enable colony expansion (default: false) |

### Confinement Protocol

The replication layer requires explicit authorization:

```yaml
# config/confinement.yaml
replication:
  enabled: false  # Must be explicitly enabled
  max_colonies: 1
  require_sudo: true
  allowed_networks:
    - localhost
    - 10.0.0.0/8
```

## Safety Protocols

The Octopi neural mesh includes multiple safety layers to prevent uncontrolled replication:

1. **Confinement Protocol**: Replication requires explicit configuration
2. **Admin Authentication**: Colony expansion requires sudo access
3. **Network Isolation**: Replication limited to configured networks
4. **Token Budgeting**: Each agent operates within defined token limits
5. **Emergency Shutdown**: All agents can be terminated remotely
